# Claude AI 직접 플로우차트 생성 프롬프트

Claude AI에 입력하여 직접 플로우차트를 그려달라고 요청하는 프롬프트입니다. Mermaid 코드가 아닌 시각적 플로우차트 이미지를 생성합니다.

---

## 🎯 전체 통합 플로우차트 프롬프트

```
당신은 세계 최고의 플로우차트 전문가입니다. Episode 서비스의 전체 사용자 여정을 플로우차트로 그려주세요.

## 플로우차트 스타일 요구사항:
1. 표준 플로우차트 기호 사용:
   - 타원형: 시작/종료
   - 사각형: 프로세스/액션
   - 다이아몬드: 조건 분기/판단
   - 평행사변형: 입력/출력
   - 둥근 사각형: 알림/메시지
2. 액터가 있는 경우: 왼쪽에 액터 아이콘 배치
3. 색상 구분:
   - 시작/종료: 어두운 회색
   - 일반 프로세스: 흰색
   - 조건 분기: 흰색 다이아몬드
   - 에러/알림: 노란색 또는 빨간색
4. 화살표에 라벨 표시 (Yes/No, 성공/실패 등)
5. 병렬 프로세스는 상하로 배치

## Episode 서비스 개요:
마인드맵 기반 취업준비 서비스로, 사용자의 경험을 구조화하고 STAR 방식으로 자기소개서를 작성할 수 있도록 도와주는 서비스입니다.

## 전체 플로우:

### 진입점
- 시작: 랜딩 페이지 (/) 또는 공유 링크 (/share/[nodeId])

### 랜딩 페이지 플로우
1. 랜딩 페이지 진입
2. 로그인 상태 확인 (다이아몬드)
   - 로그인 안됨 → 서비스 소개 표시 → 기능 소개 페이지 이동 가능 → 시작하기 버튼 → 로그인 페이지
   - 로그인 됨 → 프로젝트 목록 페이지로 리다이렉트
3. OAuth 코드가 URL에 있으면 → OAuth 콜백 페이지로 리다이렉트

### 인증 플로우
1. 로그인 페이지 진입
2. 로그인 상태 확인 (다이아몬드)
   - 이미 로그인됨:
     - returnUrl 있음 → returnUrl로 이동
     - returnUrl 없음:
       - 프로젝트 있음 → 프로젝트 목록
       - 프로젝트 없음 → 배지 선택 페이지
   - 로그인 안됨 → 카카오 로그인 버튼 표시
3. 카카오 로그인 클릭 → 카카오 OAuth 페이지로 리다이렉트
4. 카카오 로그인 완료 → OAuth 콜백 (/auth/callback?code=...)
5. 인증 코드 처리
6. Supabase Auth로 사용자 정보 조회/생성 (다이아몬드)
   - 성공 → localStorage에 사용자 정보 저장 → returnUrl 있으면 returnUrl로, 없으면 프로젝트 있으면 프로젝트 목록, 없으면 배지 선택 페이지
   - 실패 → 에러 메시지 표시 (알림) → 로그인 페이지 유지

### 프로젝트 관리 플로우
1. 프로젝트 목록 페이지 진입
2. 로그인 확인 (다이아몬드)
   - 로그인 안됨 → 로그인 페이지로 리다이렉트
   - 로그인 됨 → 프로젝트 목록 조회
3. 프로젝트 존재 여부 확인 (다이아몬드)
   - 프로젝트 있음 → 프로젝트 카드 표시 → 프로젝트 클릭 → 마인드맵 편집 페이지
   - 프로젝트 없음 → 빈 상태 표시 → 새 프로젝트 버튼 → 배지 선택 페이지
4. 배지 선택 페이지
   - 8가지 배지 표시
   - 배지 선택 (최소 1개 이상)
   - 기타 배지 선택 시 → 커스텀 라벨 입력 (평행사변형)
   - 완료 버튼 클릭
5. 프로젝트 초기화
   - UUID 프로젝트 ID 생성
   - 프로젝트명 자동 생성
   - 중앙 노드 생성
   - 선택한 배지 수만큼 대분류 노드 생성
   - Supabase DB에 저장
   - 마인드맵 편집 페이지로 이동

### 마인드맵 편집 플로우
1. 마인드맵 편집 페이지 진입
2. 최초 진입 여부 확인 (다이아몬드)
   - 최초 진입 → 온보딩 튜토리얼 표시 여부 확인 (다이아몬드)
     - 튜토리얼 미표시 → 4단계 튜토리얼 오버레이 → 건너뛰기/완료 → 마인드맵 표시
     - 튜토리얼 이미 표시됨 → 마인드맵 표시
   - 재진입 → 마인드맵 표시

### 노드 조작 플로우
1. 노드 클릭 → 노드 선택 → 사이드바에 AI 어시스턴트 자동 표시
2. 노드 더블클릭 → 인라인 편집 모드 → 라벨 수정 (평행사변형) → Enter/포커스 아웃 → 저장
3. 노드 우클릭 → 컨텍스트 메뉴 표시:
   - 하위 노드 추가 → 방향 선택 → 새 노드 생성 → 편집 모드
   - 편집 → 인라인 편집 모드
   - STAR 정리하기 → 사이드바 STAR 탭 열기
   - 공유하기 → 공유 링크 생성
   - 공유 해제 → 공유 정보 삭제
   - 새 탭에서 열기 → 새 탭 생성
   - 삭제 → 삭제 확인 다이얼로그 (알림) → 확인 → 노드 및 하위 노드 삭제
4. 노드 드래그 → 위치 이동 → 좌표 저장
5. 노드에 마우스 오버 → '+' 버튼 표시 → 클릭 → 하위 노드 추가

### 사이드바 기능 플로우

#### 공백 진단하기 탭
1. 공백 진단하기 버튼 클릭 → 사이드바 열기 → 공백 진단 탭
2. 부족 역량확인 서브 탭:
   - 기업 선택 → 직무 목록 표시
   - 직무 선택 → 문항 목록 표시
   - 문항별 답변 어려움 선택 (쉬움/보통/어려움) (다이아몬드)
     - 어려움 선택 → 역량 태그 추출 → 추천 인벤토리에 자동 추가
   - 모든 문항 완료 → 분석 결과 표시
3. 추천 인벤토리 서브 탭:
   - 역량 태그 목록 조회
   - 역량 태그를 마인드맵 노드로 드래그 → 노드 추가 다이얼로그 표시 (알림)
     - 노드 이름 입력 (평행사변형) → 생성하기 → 새 노드 생성 → AI 어시스턴트 자동 열기
     - 취소 → 다이얼로그 닫기

#### 어시스턴트 탭
1. 어시스턴트 버튼 클릭 또는 노드 선택 시 → 사이드바 열기 → 어시스턴트 탭
2. STAR 단계별 대화 진행:
   - Situation(상황) 단계 → 질문 및 답변 수집 → 단계 완료
   - Task(과제) 단계 → 질문 및 답변 수집 → 단계 완료
   - Action(행동) 단계 → 질문 및 답변 수집 → 단계 완료
   - Result(결과) 단계 → 질문 및 답변 수집 → 단계 완료
3. 모든 단계 완료 확인 (다이아몬드)
   - 완료 → STAR 초안 생성 버튼 활성화 → STAR 초안 생성 → STAR 에디터에 자동 로드
   - 미완료 → 계속 대화 진행

#### STAR 정리하기 탭
1. STAR 정리하기 버튼 클릭 → 사이드바 열기 → STAR 탭
2. 에피소드 노드 목록 조회 → 카드 형식으로 표시
3. 에피소드 카드 클릭 → STAR 에디터 표시
4. STAR 편집:
   - Situation, Task, Action, Result 각각 편집 (평행사변형)
   - 글자 수 실시간 표시 및 제한 검증
   - 자동 저장
   - 복사 버튼 → 클립보드 복사

### 공유 링크 플로우
1. 공유 링크 생성:
   - 노드 우클릭 → 공유하기 선택
   - 공유할 노드 및 하위 노드 정보 수집
   - STAR 에셋 포함 여부 선택
   - Supabase shared_nodes 테이블에 저장
   - 공유 링크 생성: /share/{nodeId}
   - 클립보드에 자동 복사
   - 복사 완료 토스트 메시지 표시 (알림)

2. 공유 링크 접근 (/share/[nodeId]):
   - nodeId 유효성 검증 (다이아몬드)
     - 유효하지 않음 → 에러 메시지 (알림) → 404 페이지
     - 유효함 → 공유 노드 정보 조회
   - 공유 노드 존재 여부 확인 (다이아몬드)
     - 공유 노드 없음 → 에러 메시지 (알림) → 404 페이지
     - 공유 노드 있음 → 노드 및 하위 노드 로드 → 로그인 상태 확인 (다이아몬드)
       - 로그인 안됨 → 읽기 전용 모드
         - 마인드맵 표시 (읽기 전용)
         - 공유한 사용자 정보 표시
         - 읽기 전용 안내 오버레이 표시 (알림)
       - 로그인 됨 → 편집 모드 가능
         - 노드 추가/수정/삭제 가능
         - 노드 위치 이동 가능
         - STAR 작성 가능
         - 변경 사항을 원본 프로젝트에 반영

### 아카이브 플로우
1. 아카이브 페이지 진입
2. 로그인 확인 (다이아몬드)
   - 로그인 안됨 → 로그인 페이지로 리다이렉트
   - 로그인 됨 → 모든 프로젝트의 에피소드 노드 조회
3. 에피소드 목록 테이블 형식으로 표시
4. 검색 기능:
   - 검색어 입력 (평행사변형) → 에피소드 라벨, STAR 내용, 태그에서 검색 → 실시간 필터링
5. 필터링 기능:
   - 대분류(배지)별 필터링
   - 역량 태그별 필터링
   - 회사명별 필터링
6. 에피소드 편집:
   - STAR 내용 셀 클릭 → 인라인 편집 모드 → 수정 (평행사변형) → 저장
7. 에피소드 삭제:
   - 삭제 버튼 클릭 → 삭제 확인 다이얼로그 (알림) → 확인 → 에피소드 및 STAR 에셋 삭제

### 대시보드 플로우
1. 대시보드 페이지 진입
2. 로그인 확인 (다이아몬드)
   - 로그인 안됨 → 로그인 페이지로 리다이렉트
   - 로그인 됨 → 모든 프로젝트의 STAR 자기소개서 목록 조회
3. STAR 목록 카드 형식으로 표시
4. 검색 기능:
   - 검색어 입력 (평행사변형) → 제목, 내용, 회사명, 역량 태그에서 검색 → 실시간 필터링
5. STAR 상세 조회:
   - STAR 카드 클릭 → 상세 다이얼로그 표시
   - 복사 버튼 → 클립보드 복사
6. STAR 삭제:
   - 삭제 버튼 클릭 → 삭제 확인 다이얼로그 (알림) → 확인 → STAR 에셋 삭제

### 종료
- 모든 플로우는 "끝" 또는 "메인화면"으로 종료

## 플로우차트 그리기 요청:
위의 모든 플로우를 하나의 큰 플로우차트로 그려주세요. 
- 각 주요 기능 영역을 명확히 구분
- 조건 분기는 다이아몬드로 표시
- 입력/출력은 평행사변형으로 표시
- 알림/메시지는 둥근 사각형으로 표시
- 화살표에 Yes/No, 성공/실패 등 라벨 표시
- 병렬 프로세스는 상하로 배치
- 색상으로 기능 영역 구분 (인증: 파란색, 프로젝트: 초록색, 마인드맵: 보라색, 공유: 주황색, 아카이브: 회색, 대시보드: 노란색)
```

### 프롬프트 2: 프로젝트 생성 플로우

```
Episode 서비스의 프로젝트 생성 플로우를 플로우차트로 그려주세요.

## 스타일:
```
