# Episode 서비스 용어 정리

| 용어 | 내용 | 비고 |
|------|------|------|
| 마인드맵 | 취업 준비과정에 활용할 활동내용을 계층형 구조로 체계적으로 정리하는 도구 | - |
| 노드 | 마인드맵을 구성하는 요소 단위 | 각 노드는 텍스트 내용을 가지며, 다른 노드와 연결 관계를 가짐 |
| 노드 계층 | 마인드맵의 계층 구조 | 개인 마인드맵: 중심 노드 - 경험 카테고리 - 경험 - 에피소드<br>팀 마인드맵: 중심 노드 - 경험 - 에피소드 |
| 중심 노드 | 마인드맵의 최상위 노드로, 모든 하위 노드의 루트가 되는 노드 | 마인드맵 프로젝트의 주제를 나타냄 |
| 경험 카테고리 | 개인 마인드맵에서 경험을 분류하는 상위 카테고리 | 배지(인턴, 동아리, 프로젝트 등)로 표현됨 |
| 경험 | 구체적인 활동이나 프로젝트를 나타내는 노드 | 예: "웹 개발 프로젝트", "디자인 시스템 구축" |
| 에피소드 | 경험 내에서 발생한 구체적인 상황이나 사건을 나타내는 노드 | STAR 형식으로 정리됨 |
| STAR | 에피소드 정리에 활용되는 방법론 | 특정 에피소드를 S(Situation)-T(Task)-A(Action)-R(Result) 항목 단위로 정리 |
| Situation | STAR의 첫 번째 항목으로, 에피소드가 발생한 상황이나 배경 | - |
| Task | STAR의 두 번째 항목으로, 해결해야 할 과제나 목표 | - |
| Action | STAR의 세 번째 항목으로, 실제로 수행한 행동이나 조치 | - |
| Result | STAR의 네 번째 항목으로, 행동의 결과나 성과 | - |
| 에피소드 보관함 | 마인드맵에서 정리한 모든 에피소드를 STAR 형식으로 한눈에 조회하고 편집할 수 있는 통합 관리 페이지 | - |
| 프로젝트 | 하나의 마인드맵 단위 | 사용자는 여러 프로젝트를 생성하고 관리할 수 있음 |
| 배지 | 개인 마인드맵 생성 시 선택하는 경험 카테고리 유형 | 인턴, 동아리, 프로젝트, 대회, 봉사, 아르바이트, 기타 등 |
| 개인 마인드맵 | 본인 개인의 경험을 정리하기 위해 활용하는 비공개 마인드맵 | 공유 기능 없음, 개인 전용 |
| 팀 마인드맵 | 여러 사용자가 동시에 편집할 수 있으며 공유 기능이 활성화된 마인드맵 | 공유 링크를 통해 여러 사용자가 함께 편집 가능 |
| 공유 링크 | 팀 마인드맵을 다른 사용자와 공유하기 위한 URL | 링크를 아는 모든 사용자가 접근 가능 |
| 실시간 협업 | 팀 마인드맵에서 여러 사용자가 동시에 편집하는 기능 | 현재 편집 중인 사용자 목록을 실시간으로 표시 |
| 읽기 전용 모드 | 공유 링크로 접근한 비로그인 사용자 또는 권한이 없는 사용자가 볼 수 있는 모드 | 노드/레이아웃/설정 편집 불가 |
| 기출문항 | 공채 모집을 진행한 적이 있는 기업들에서 출제된 자기소개서 문항 | 본 프로덕트 MVP에서는 특정 기준으로 상위 기업의 최근 기출문항을 활용함 |
| 기출문항 셀프진단 | 희망 직무별 기업의 최근 기출 자기소개서 문항을 통해 추가 정리가 필요한 역량 범주를 분석하는 기능 | 별도 진단 페이지 또는 마인드맵 캔버스 내에서 실행 가능 |
| 역량 | 자기소개서나 면접에서 요구되는 능력이나 자질 | 예: 리더십, 커뮤니케이션, 문제 해결 능력 등 |
| 역량 빈틈 | 기출문항 셀프진단 결과로 도출된, 사용자가 보충하면 좋을 역량 범주 | "모르겠음"으로 답변한 질문들을 기반으로 분석됨 |
| 역량 필터 | 에피소드의 STAR 태그 기반 역량 필터링 기능 | 필터링된 노드가 하이라이트 표시됨 |
| 추천 인벤토리 | 기출문항 셀프진단 결과로 도출된 역량 빈틈 키워드를 마인드맵 사이드바에서 조회할 수 있는 기능 | 마인드맵 캔버스에 상시적으로 보여줌 |
| 강점/역량 태그 | 에피소드에 태그된 역량 키워드 | STAR 작성 시 추가 및 관리 가능 |
| 마인드맵 캔버스 | 마인드맵 노드들이 표시되고 편집되는 화면 영역 | 드래그, 줌, 팬 등의 조작이 가능함 |
| 노드 연결점 | 노드의 상하좌우에 표시되는 연결 지점 | 클릭 시 하위 노드 추가 가능 |
| 자유 노드 | 캔버스 빈 공간에 생성된 독립적인 노드 | 드래그하여 기존 노드와 연결 가능 |
| 레이아웃 | 마인드맵 노드들의 배치 방식 | 원형 레이아웃, 트리형 레이아웃, 브랜치 레이아웃 |
| 원형 레이아웃 | 중심 노드를 기준으로 원형 배치하는 레이아웃 | - |
| 트리형 레이아웃 | 좌우 대칭 트리 구조로 배치하는 레이아웃 | - |
| 브랜치 레이아웃 | 중심에서 좌우로 뻗어나가는 직각 브랜치 배치 레이아웃 | - |
| 커서 모드 | 마인드맵 캔버스의 조작 모드 | 선택 모드: 노드 선택 및 편집 가능<br>이동 모드: 캔버스 팬(pan) 가능 |
| 화면 맞춤 | 모든 노드를 화면에 맞게 자동으로 배치하는 기능 | - |
| 탭 관리 | 여러 마인드맵 프로젝트를 동시에 탭으로 열고 관리하는 기능 | 새로고침 후에도 열려있던 탭 자동 복원 |
| 툴바 | 마인드맵 캔버스 좌측에 위치한 도구 모음 | 전체 보기, 노드 추가, 그리드 토글, 내보내기 등 |
| 사이드바 | 마인드맵 캔버스 우측에 위치한 패널 | 기출문항 셀프진단, STAR 정리 기능 제공 |
| 온보딩 | 최초 마인드맵 생성 시 표시되는 튜토리얼 | 4단계로 구성됨 |
| 프로젝트 타입 | 마인드맵 프로젝트의 유형 | 개인 마인드맵 또는 팀 마인드맵 중 선택, 생성 후 변경 불가 |
| 경험 기간 | 경험 노드의 시작일 및 종료일 정보 | 에피소드 보관함에서 편집 가능 |
| 내보내기 | 마인드맵을 이미지(PNG) 또는 PDF 형식으로 저장하는 기능 | - |
| 검색 | 노드 내용에 포함된 키워드를 검색하는 기능 | 검색된 노드가 하이라이트 표시됨 |
| 그리드 | 마인드맵 캔버스에 표시되는 격자선 | 시각적 정렬을 돕는 보조 도구 |
| 브랜치별 색상 | 경험 노드를 기준으로 하위 노드들에 자동 할당되는 색상 | 같은 경험 브랜치의 노드들은 동일한 색상 사용 |
| 공유 해제 | 팀 마인드맵의 공유 기능을 비활성화하는 기능 | 공유 링크 무효화 및 기존 접근자 권한 차단 |
| 편집자 표시 | 현재 마인드맵을 편집 중인 사용자 목록을 표시하는 기능 | 팀 마인드맵에서만 표시, 실시간 업데이트 |
| 프로젝트 목록 | 사용자가 생성한 모든 마인드맵 프로젝트를 조회하는 페이지 | 개인 마인드맵과 팀 마인드맵을 탭으로 구분 |
| 랜딩 페이지 | 서비스의 메인 홈 화면 | 로그인하지 않은 사용자도 접근 가능, 서비스 소개 및 기능 안내 |
| OAuth 콜백 | 카카오 로그인 인증 후 리다이렉트되는 페이지 | 인증 코드 처리 및 사용자 정보 저장 |
