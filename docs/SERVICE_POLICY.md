## 서비스 개요

### 1.1 서비스 명칭
**Episode** (에피소드)

### 1.2 서비스 성격
마인드맵 기반 취업 준비 지원 서비스

### 1.3 주요 기능
1. **기출문항 셀프진단**: 희망 직무별 기업의 최근 기출 자기소개서 문항을 통한 역량 빈틈 분석
2. **마인드맵 기반 활동 정리**: 계층형 마인드맵을 통한 경험 구조화 및 관리
3. **에피소드 보관함**: STAR 형식으로 작성된 모든 에피소드의 통합 관리
4. **실시간 협업**: 팀 마인드맵에서 여러 사용자의 동시 편집 지원
5. **데이터 내보내기**: 마인드맵 이미지/PDF 저장 기능

### 1.4 서비스 유형
- **개인 마인드맵**: 개인 사용자의 경험 정리를 위한 비공개 마인드맵
- **팀 마인드맵**: 여러 사용자가 함께 편집할 수 있는 공유 마인드맵

### 1.5 관련 법령
- 개인정보 보호법
- 정보통신망 이용촉진 및 정보보호 등에 관한 법률
- 전자상거래 등에서의 소비자 보호에 관한 법률
- 저작권법

## 회원 가입 및 인증 정책

### 2.1 회원 가입 자격
- 만 14세 이상의 개인
- 본인 명의의 카카오 계정 보유자
- 서비스 이용약관에 동의한 자

### 2.2 회원 가입 방법
- **카카오 OAuth 인증**: 카카오 계정을 통한 간편 로그인
- 회원 가입과 동시에 로그인 처리됨
- 최초 로그인 시 자동으로 회원 계정 생성

### 2.3 회원 정보 수집 항목
회원 가입 시 다음 정보를 수집합니다:
- **필수 항목**: 
  - 카카오 계정 ID (provider_user_id)
  - 인증 제공자 (provider: 'kakao')
  - 이름 (카카오 프로필에서 제공되는 닉네임 또는 이름)
  - 이메일 (카카오 계정 이메일, 제공 동의 시)
- **자동 수집 항목**:
  - 회원 가입 일시
  - 최종 로그인 일시
  - 서비스 이용 기록

### 2.4 회원 가입 제한
다음의 경우 회원 가입이 제한될 수 있습니다:
- 만 14세 미만인 경우
- 타인의 명의를 도용한 경우
- 서비스 이용약관을 위반한 이력이 있는 경우
- 기타 법령 및 서비스 정책에 위배되는 경우

### 2.5 로그인 정책
- **로그인 방법**: 카카오 OAuth 인증만 지원
- **세션 관리**: 
  - Supabase Auth를 통한 세션 관리
  - 브라우저 세션 및 로컬 스토리지를 통한 인증 상태 유지
- **자동 로그인**: 브라우저 세션 유지 시 자동 로그인 상태 유지
- **로그인 실패**: 카카오 인증 실패 시 에러 메시지 표시 및 재시도 안내

### 2.6 회원 정보 변경
- 이름, 이메일 등 회원 정보는 카카오 계정 정보와 연동됨
- 회원 정보 변경은 카카오 계정 설정에서 변경 후 서비스 재로그인 필요

## 개인정보 처리 정책

### 3.1 개인정보 수집 및 이용 목적
- **서비스 제공**: 마인드맵 생성, 관리, 공유 기능 제공
- **회원 관리**: 회원 식별, 본인 확인, 부정 이용 방지
- **서비스 개선**: 이용 통계 분석, 서비스 품질 개선
- **고객 지원**: 문의 사항 처리, 공지 사항 전달

### 3.2 개인정보 수집 항목
- **회원 정보**: 이름, 이메일, 카카오 계정 ID, 인증 제공자
- **서비스 이용 정보**: 
  - 마인드맵 프로젝트 데이터 (제목, 노드 구조, 내용)
  - STAR 에셋 데이터 (Situation, Task, Action, Result, 역량 태그)
  - 공백 진단 결과 데이터
  - 에피소드 아카이브 데이터
- **자동 수집 정보**: 
  - IP 주소, 쿠키, 접속 로그
  - 기기 정보, 브라우저 정보
  - 서비스 이용 기록

### 3.3 개인정보 보관 기간
- **회원 정보**: 회원 탈퇴 시까지 보관 (단, 법령에 따라 보관이 필요한 경우 해당 기간 동안 보관)
- **서비스 이용 정보**: 회원 탈퇴 시 삭제 또는 익명화 처리
- **법령에 따른 보관**: 
  - 전자상거래법: 계약 및 청약철회 기록 5년
  - 통신비밀보호법: 로그 기록 3개월

### 3.4 개인정보 제3자 제공
- **제3자 제공 없음**: 원칙적으로 개인정보를 제3자에게 제공하지 않음
- **예외 사항**:
  - 사용자가 공유 기능을 통해 명시적으로 공유한 경우
  - 법령에 따라 제공이 의무화된 경우
  - 서비스 제공을 위해 필요한 최소한의 범위 내에서만 제공

### 3.5 개인정보 처리 위탁
- **Supabase (인프라 제공)**: 
  - 데이터베이스 호스팅 및 인증 서비스
  - 위치: 해외 (데이터 저장 위치는 Supabase 정책에 따름)
  - 위탁 기간: 서비스 제공 기간 동안
- **카카오 (인증 서비스)**: 
  - OAuth 인증 서비스 제공
  - 위치: 국내
  - 위탁 기간: 인증 처리 시에만

### 3.6 개인정보 보호 조치
- **기술적 조치**: 
  - SSL/TLS 암호화 통신
  - 데이터베이스 암호화
  - 접근 권한 관리 (RLS: Row Level Security)
- **관리적 조치**: 
  - 개인정보 처리 담당자 최소화
  - 정기적인 보안 점검
  - 개인정보 처리 방침 정기 검토 및 개선

### 3.7 개인정보 열람, 정정, 삭제
- 사용자는 언제든지 본인의 개인정보를 열람, 정정, 삭제할 수 있음
- 회원 탈퇴 시 모든 개인정보 및 서비스 데이터 삭제 처리
- 개인정보 열람/정정/삭제 요청은 서비스 내 설정 메뉴 또는 고객 지원을 통해 가능

## 서비스 이용 정책

### 4.1 서비스 이용 범위
- **무료 서비스**: 기본 마인드맵 생성 및 관리 기능
- **제한 사항**: 
  - 서비스 이용량 제한 (프로젝트 개수, 노드 개수 등)은 추후 정책에 따라 변경 가능
  - 현재는 별도의 이용량 제한 없음

### 4.2 마인드맵 프로젝트 관리
- **프로젝트 생성**:
  - 개인 마인드맵: 배지(경험 카테고리) 선택 후 생성
  - 팀 마인드맵: 경험 노드 직접 입력 후 생성
  - 프로젝트 타입은 생성 후 변경 불가
- **프로젝트 수정**:
  - 프로젝트 소유자만 수정 가능
  - 제목, 노드 구조, 내용 수정 가능
- **프로젝트 삭제**:
  - 프로젝트 소유자만 삭제 가능
  - 삭제 시 모든 노드 및 STAR 데이터 영구 삭제
  - 삭제된 데이터는 복구 불가

### 4.3 노드 관리 정책
- **노드 생성**:
  - 무제한 노드 생성 가능
  - 계층 구조 자동 적용 (중심 노드 → 카테고리 → 경험 → 에피소드)
- **노드 편집**:
  - 노드 더블클릭 또는 우클릭 메뉴를 통한 편집
  - 인라인 편집 지원
- **노드 삭제**:
  - 노드 삭제 시 하위 노드까지 함께 삭제됨
  - 삭제된 노드는 복구 불가

### 4.4 STAR 데이터 관리
- **STAR 작성**:
  - 에피소드 노드에 STAR 형식으로 내용 작성
  - Situation, Task, Action, Result 각 항목별 입력
  - 역량 태그 추가 및 관리
- **STAR 수정/삭제**:
  - 작성자 또는 프로젝트 소유자만 수정/삭제 가능
  - 에피소드 아카이브 페이지에서 일괄 편집 가능

### 4.5 데이터 내보내기
- **이미지 내보내기**: PNG 형식으로 마인드맵 저장
- **PDF 내보내기**: PDF 형식으로 마인드맵 저장
- **데이터 백업**: 
  - 현재 자동 백업 기능 없음
  - 사용자가 직접 내보내기 기능을 통해 백업 권장

### 4.6 서비스 이용 제한
다음의 경우 서비스 이용이 제한될 수 있습니다:
- 서비스 이용약관 위반
- 타인의 권리를 침해하는 행위
- 불법적이거나 부적절한 콘텐츠 작성
- 시스템에 악영향을 미치는 행위
- 기타 법령 및 서비스 정책 위반

## 콘텐츠 및 데이터 관리 정책

### 5.1 사용자 생성 콘텐츠 (UGC)
- 사용자가 작성한 모든 콘텐츠(마인드맵, 노드 내용, STAR 데이터 등)의 저작권은 사용자에게 있음
- 사용자는 본인이 작성한 콘텐츠에 대한 책임을 짐

### 5.2 콘텐츠 작성 규칙
다음의 콘텐츠는 작성이 금지됩니다:
- 타인의 저작권을 침해하는 콘텐츠
- 명예훼손, 모욕, 비방에 해당하는 콘텐츠
- 불법적이거나 부적절한 콘텐츠
- 개인정보(본인 외 타인의 정보)를 무단으로 수집하거나 공개하는 콘텐츠
- 스팸, 광고, 홍보성 콘텐츠
- 기타 법령 및 서비스 정책에 위배되는 콘텐츠

### 5.3 콘텐츠 관리
- **콘텐츠 검토**: 
  - 원칙적으로 사전 검토 없이 게시됨
  - 신고 접수 시 검토 및 조치
- **콘텐츠 삭제**: 
  - 위반 콘텐츠 발견 시 즉시 삭제
  - 사용자 신고 접수 시 검토 후 삭제 가능
- **콘텐츠 수정 요청**: 
  - 사용자가 직접 수정 가능
  - 관리자에 의한 수정 요청 시 사용자에게 통지

### 5.4 데이터 보관 및 삭제
- **데이터 보관**: 
  - 사용자 데이터는 회원 탈퇴 시까지 보관
  - 공유된 마인드맵의 경우 공유 해제 시까지 보관
- **데이터 삭제**: 
  - 회원 탈퇴 시 모든 데이터 즉시 삭제
  - 프로젝트 삭제 시 관련 모든 데이터 즉시 삭제
  - 삭제된 데이터는 복구 불가

### 5.5 데이터 백업
- **자동 백업**: 현재 자동 백업 기능 없음
- **수동 백업**: 사용자가 내보내기 기능을 통해 직접 백업 권장
- **서비스 장애 시**: 
  - 서비스 장애로 인한 데이터 손실에 대한 책임은 서비스 제공자가 부담
  - 단, 사용자의 고의 또는 과실로 인한 손실은 제외

## 공유 및 협업 정책

### 6.1 공유 기능
- **개인 마인드맵**: 공유 기능 없음 (개인 전용)
- **팀 마인드맵**: 공유 기능 활성화 가능

### 6.2 공유 설정
- **공유 활성화**: 
  - 팀 마인드맵에서만 공유 기능 사용 가능
  - 프로젝트 소유자만 공유 설정 변경 가능
  - 공유 활성화 시 공유 링크 자동 생성
- **공유 링크**: 
  - 형식: `https://[도메인]/mindmap/[projectId]`
  - 링크를 아는 모든 사용자가 접근 가능
  - 링크 공유 시 주의 필요

### 6.3 공유 접근 권한
- **비로그인 사용자**: 
  - 읽기 전용 모드
  - 노드/레이아웃/설정 편집 불가
  - "읽기 전용 모드 – 편집하려면 로그인" 배너 표시
- **로그인 사용자**: 
  - 프로젝트 소유자: 전체 편집 가능
  - 공유 링크 접근자: 편집 가능 (현재 권한 모델 단순화)
  - 추후 역할(Role) 기반 권한 관리 도입 예정

### 6.4 실시간 협업
- **동시 편집**: 
  - 여러 사용자가 동시에 마인드맵 편집 가능
  - 실시간 변경 사항 동기화
- **편집자 표시**: 
  - 현재 편집 중인 사용자 목록 표시
  - 화면 상단에 접속자 아바타/이니셜 표시
  - "나" 표시로 본인 구분
- **편집자 판단 기준**: 
  - `active_editors` 테이블의 `last_seen`이 마지막 5분 이내인 사용자
  - 5분 이상 비활성 시 자동으로 편집자 목록에서 제거

### 6.5 공유 해제
- **공유 해제**: 
  - 프로젝트 소유자만 공유 해제 가능
  - 공유 해제 시 공유 링크 무효화
  - 기존 접근자들의 접근 권한 즉시 차단

### 6.6 공유 데이터 보호
- **데이터 보안**: 
  - 공유된 마인드맵도 개인정보 보호 정책 적용
  - 공유 링크 노출 시 데이터 유출 위험 있음
  - 사용자는 공유 링크 관리에 주의 필요
- **공유 책임**: 
  - 공유 링크를 통한 데이터 접근에 대한 책임은 공유한 사용자에게 있음
  - 서비스 제공자는 공유 링크 관리에 대한 책임을 지지 않음

## 서비스 이용 제한 및 계정 관리

### 8.1 서비스 이용 제한 사유
다음의 경우 서비스 이용이 제한될 수 있습니다:
- 서비스 이용약관 위반
- 타인의 권리를 침해하는 행위
- 불법적이거나 부적절한 콘텐츠 작성
- 시스템에 악영향을 미치는 행위 (DDoS, 스크래핑 등)
- 기타 법령 및 서비스 정책 위반

### 8.2 이용 제한 조치
- **경고**: 1차 위반 시 경고 조치
- **일시 정지**: 반복 위반 시 일정 기간 서비스 이용 정지
- **영구 정지**: 중대한 위반 시 계정 영구 정지
- **데이터 삭제**: 위반 콘텐츠 및 관련 데이터 삭제

### 8.3 계정 정지 시 조치
- 계정 정지 시 서비스 이용 불가
- 정지 기간 동안 데이터 보관 (정지 해제 시 복구 가능)
- 영구 정지 시 모든 데이터 삭제

### 8.4 회원 탈퇴
- **탈퇴 방법**: 서비스 내 설정 메뉴에서 탈퇴 신청
- **탈퇴 처리**: 
  - 탈퇴 신청 즉시 모든 데이터 삭제
  - 삭제된 데이터는 복구 불가
- **탈퇴 제한**: 
  - 공유 중인 마인드맵이 있는 경우 탈퇴 전 공유 해제 필요
  - 미결제 금액이 있는 경우 결제 완료 후 탈퇴 가능

## 지적재산권 정책

### 9.1 서비스 지적재산권
- 서비스의 모든 디자인, 로고, UI/UX, 소스코드 등은 서비스 제공자의 지적재산권에 속함
- 사용자는 서비스 제공자의 사전 허가 없이 서비스를 복제, 배포, 수정할 수 없음

### 9.2 사용자 콘텐츠 지적재산권
- 사용자가 작성한 모든 콘텐츠의 저작권은 사용자에게 있음
- 사용자는 본인이 작성한 콘텐츠를 자유롭게 이용할 수 있음
- 서비스 제공자는 사용자 콘텐츠를 서비스 제공 목적으로만 사용

### 9.3 제3자 지적재산권
- 서비스에서 사용하는 제3자 라이브러리, 폰트, 아이콘 등은 각각의 라이선스에 따름
- 사용자는 제3자 지적재산권을 침해하는 콘텐츠를 작성할 수 없음

### 9.4 저작권 침해 신고
- 저작권 침해 신고는 고객 지원을 통해 접수
- 신고 접수 시 검토 후 조치
- 반복적인 저작권 침해 시 계정 정지 조치 가능

## 면책 및 책임 제한

### 10.1 서비스 제공자의 면책
서비스 제공자는 다음의 경우 책임을 지지 않습니다:
- 천재지변, 전쟁, 폭동, 해킹, 바이러스 등 불가항력으로 인한 서비스 중단
- 사용자의 고의 또는 과실로 인한 데이터 손실
- 사용자가 작성한 콘텐츠로 인한 분쟁
- 제3자의 불법적 행위로 인한 피해
- 통신사업자의 통신 장애로 인한 서비스 이용 불가

### 10.2 서비스 품질 보장
- 서비스 제공자는 서비스의 지속적이고 안정적인 제공을 위해 노력함
- 단, 서비스의 품질이나 성능에 대한 보장은 하지 않음
- 서비스 중단, 오류, 데이터 손실 등에 대한 책임은 제한됨

### 10.3 제3자 서비스 연동
- 카카오 OAuth, Supabase 등 제3자 서비스 연동 시 해당 서비스의 장애나 변경으로 인한 영향에 대한 책임은 지지 않음
- 제3자 서비스의 정책 변경으로 인한 서비스 이용 제한에 대한 책임은 지지 않음

## 서비스 변경 및 종료 정책

### 11.1 서비스 변경
- 서비스 제공자는 서비스 개선을 위해 서비스 내용을 변경할 수 있음
- 중대한 변경 사항은 사전 공지
- 변경 사항에 대한 사용자의 동의는 서비스 이용을 통해 간주됨

### 11.2 서비스 중단
- **일시 중단**: 
  - 시스템 점검, 업데이트 등을 위한 일시 중단
  - 사전 공지 후 진행 (긴급 상황 제외)
- **장기 중단**: 
  - 서비스 개편, 인프라 변경 등을 위한 장기 중단
  - 최소 7일 전 공지
  - 중단 기간 동안 데이터 보관

### 11.3 서비스 종료
- **서비스 종료 사유**: 
  - 사업 정책 변경
  - 기술적 한계
  - 기타 불가피한 사유
- **서비스 종료 절차**: 
  - 최소 30일 전 공지
  - 사용자 데이터 내보내기 기간 제공
  - 종료일 이후 모든 데이터 삭제

### 11.4 데이터 이전
- 서비스 종료 시 사용자 데이터 이전 지원
- 사용자가 직접 내보내기 기능을 통해 데이터 백업 권장
- 서비스 종료 후 데이터 복구 불가
