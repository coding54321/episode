## 마인드맵 프로젝트 관리 정책

### 1.1 프로젝트 생성 정책

#### 1.1.1 프로젝트 타입 선택
- **개인 마인드맵**
  - 개인 사용자의 경험을 정리하기 위한 비공개 마인드맵
  - 공유 기능 없음
  - 계층 구조: 중심 노드 → 경험 카테고리 → 경험 → 에피소드
  - 배지(경험 카테고리) 선택 필수 (최소 1개 이상)
  - 배지 종류: 인턴, 동아리, 프로젝트, 대회, 봉사, 아르바이트, 기타
  - "기타" 배지 선택 시 커스텀 라벨 입력 가능

- **팀 마인드맵**
  - 여러 사용자가 동시에 편집할 수 있는 공유 마인드맵
  - 공유 기능 활성화 가능
  - 계층 구조: 중심 노드 → 경험 → 에피소드
  - 경험 노드 직접 입력 필수 (최소 1개 이상)
  - 배지 선택 불필요

#### 1.1.2 프로젝트 생성 제약사항
- 프로젝트 타입은 생성 후 변경 불가
- 프로젝트 이름은 필수 입력 항목
- 프로젝트 이름 최대 길이: 100자
- 사용자당 생성 가능한 프로젝트 개수: 제한 없음 (현재)
- 프로젝트 생성 시 자동으로 중심 노드 생성됨

#### 1.1.3 프로젝트 생성 프로세스
1. 프로젝트 타입 선택 (개인/공동)
2. 개인 마인드맵: 배지 선택 → 프로젝트 이름 입력 → 생성
3. 팀 마인드맵: 경험 노드 입력 → 프로젝트 이름 입력 → 생성
4. 생성 완료 시 마인드맵 워크스페이스로 이동
5. 최초 진입 시 온보딩 튜토리얼 자동 표시

### 1.2 프로젝트 수정 정책

#### 1.2.1 프로젝트 제목 수정
- 프로젝트 소유자만 수정 가능
- 인라인 편집 방식 지원
- 제목 더블클릭 또는 우클릭 메뉴를 통한 편집
- 최대 길이: 100자
- 빈 제목 불가

#### 1.2.2 프로젝트 내용 수정
- 노드 구조, 내용, STAR 데이터 수정 가능
- 프로젝트 타입 변경 불가
- 배지 추가/삭제 불가 (개인 마인드맵)
- 프로젝트 설정(레이아웃, 색상 등) 수정 가능

### 1.3 프로젝트 삭제 정책

#### 1.3.1 삭제 권한
- 프로젝트 소유자만 삭제 가능
- 팀 마인드맵의 경우 소유자만 삭제 가능 (팀 편집자 삭제 불가)

#### 1.3.2 삭제 프로세스
1. 프로젝트 목록에서 삭제 버튼 클릭
2. 삭제 확인 다이얼로그 표시
3. 삭제 시 모든 노드 및 STAR 데이터 영구 삭제 안내
4. 확인 시 즉시 삭제 처리
5. 삭제된 데이터는 복구 불가

#### 1.3.3 삭제 범위
- 프로젝트 정보
- 모든 노드 및 계층 구조
- 모든 STAR 데이터
- 공유 링크 무효화
- 관련 에피소드 아카이브 데이터

## 노드 관리 정책

### 2.1 노드 생성 정책

#### 2.1.1 노드 생성 방법
- **하위 노드 추가**
  - 노드 우클릭 메뉴에서 "하위 노드 추가" 선택
  - 노드 연결점(상하좌우) 클릭
  - 자동으로 계층 구조에 맞는 노드 타입 할당

- **자유 노드 생성**
  - 툴바의 "노드 추가" 버튼 클릭 후 캔버스 빈 공간 클릭
  - 캔버스 빈 공간 우클릭 메뉴에서 "노드 추가" 선택
  - 초기에는 연결선 없이 독립적으로 생성
  - 드래그하여 기존 노드와 연결 가능

#### 2.1.2 노드 계층 구조
- **개인 마인드맵**
  - Level 0: 중심 노드 (1개, 고정)
  - Level 1: 경험 카테고리 (배지별 분류)
  - Level 2: 경험 (구체적인 활동/프로젝트)
  - Level 3: 에피소드 (STAR 작성 대상)

- **팀 마인드맵**
  - Level 0: 중심 노드 (1개, 고정)
  - Level 1: 경험 (구체적인 활동/프로젝트)
  - Level 2: 에피소드 (STAR 작성 대상)

#### 2.1.3 노드 생성 제약사항
- 노드 개수 제한: 없음 (현재)
- 노드 이름 최대 길이: 200자
- 빈 노드 이름 불가
- 중심 노드는 1개만 존재 가능
- 중심 노드 삭제 불가

#### 2.1.4 노드 타입 자동 할당
- 노드 추가 시 부모 노드의 레벨에 따라 자동으로 노드 타입 할당
- 사용자가 직접 노드 타입 선택 불가
- 계층 구조에 맞지 않는 노드 타입 생성 불가

### 2.2 노드 편집 정책

#### 2.2.1 노드 내용 편집
- **인라인 편집**
  - 노드 더블클릭으로 인라인 편집 모드 진입
  - 편집 중 ESC 키로 취소, Enter 키로 저장
  - 편집 완료 시 자동 저장

- **우클릭 메뉴 편집**
  - 노드 우클릭 메뉴에서 편집 옵션 선택
  - 인라인 편집 모드로 전환

#### 2.2.2 노드 위치 이동
- **드래그 앤 드롭**
  - 노드를 드래그하여 위치 이동
  - 드롭 시 새로운 좌표(x, y) 저장
  - 부모-자식 관계는 유지 (계층 구조 보존)
  - 위치 변경 시 자동 저장

- **자유 노드 연결**
  - 자유 노드를 드래그하여 기존 노드 근처로 이동
  - 스냅 거리 내에 도달 시 가상 연결선 표시
  - 드롭 시 부모-자식 관계 생성 및 연결선 표시

#### 2.2.3 노드 편집 제약사항
- 읽기 전용 모드에서는 편집 불가
- 팀 마인드맵에서도 편집 가능 (권한 있는 사용자)
- 동시 편집 시 충돌 방지 로직 적용

### 2.3 노드 삭제 정책

#### 2.3.1 삭제 방법
- 노드 우클릭 메뉴에서 "삭제" 선택
- 삭제 확인 다이얼로그 표시 (하위 노드 포함 삭제 안내)
- 확인 시 즉시 삭제

#### 2.3.2 삭제 범위
- 선택한 노드 및 모든 하위 노드 일괄 삭제
- 관련 STAR 데이터도 함께 삭제
- 삭제된 노드는 복구 불가

#### 2.3.3 삭제 제약사항
- 중심 노드는 삭제 불가
- 읽기 전용 모드에서는 삭제 불가
- 프로젝트 소유자 또는 권한 있는 사용자만 삭제 가능

## STAR 작성 및 관리 정책

### 3.1 STAR 작성 정책

#### 3.1.1 STAR 작성 대상
- 에피소드 노드에만 STAR 작성 가능
- 경험 카테고리, 경험 노드에는 STAR 작성 불가
- 하나의 에피소드 노드당 하나의 STAR 데이터

#### 3.1.2 STAR 작성 방법
- **사이드바 STAR 에디터**
  - 에피소드 노드 선택 시 사이드바 STAR 에디터 자동 열림
  - Situation, Task, Action, Result 각 항목별 텍스트 영역 입력
  - 각 항목별 자유롭게 입력 가능

- **에피소드 보관함**
  - 에피소드 보관함 페이지에서 일괄 편집 가능
  - 3단 레이아웃 편집 뷰 제공

#### 3.1.3 STAR 항목별 정책
- **Situation (상황)**
  - 에피소드가 발생한 상황이나 배경 설명
  - 최대 길이: 제한 없음
  - 필수 입력 항목 아님

- **Task (과제)**
  - 해결해야 할 과제나 목표 설명
  - 최대 길이: 제한 없음
  - 필수 입력 항목 아님

- **Action (행동)**
  - 실제로 수행한 행동이나 조치 설명
  - 최대 길이: 제한 없음
  - 필수 입력 항목 아님

- **Result (결과)**
  - 행동의 결과나 성과 설명
  - 최대 길이: 제한 없음
  - 필수 입력 항목 아님

#### 3.1.4 역량 태그 정책
- **태그 추가**
  - STAR 작성 시 역량 태그 추가 가능
  - 태그 추가 다이얼로그에서 역량 키워드 선택
  - 역량 키워드 목록에서만 선택 가능 (직접 입력 불가)
  - 하나의 에피소드에 여러 역량 태그 추가 가능

- **태그 관리**
  - 추가된 태그는 표시 및 제거 가능
  - 태그는 에피소드 보관함에서도 관리 가능

### 3.2 STAR 수정 및 삭제 정책

#### 3.2.1 STAR 수정
- 작성자 또는 프로젝트 소유자만 수정 가능
- 사이드바 STAR 에디터 또는 에피소드 보관함에서 수정
- 수정 시 즉시 저장 (저장 버튼 클릭 시)

#### 3.2.2 STAR 삭제
- 에피소드 노드 삭제 시 관련 STAR 데이터도 함께 삭제
- STAR 내용만 삭제하는 기능 없음 (에피소드 삭제로만 가능)
- 삭제된 STAR 데이터는 복구 불가

## 공백 진단 정책

### 4.1 공백 진단 실행 정책

#### 4.1.1 진단 실행 위치
- **별도 진단 페이지**
  - `/gap-diagnosis-standalone` 경로에서 독립적으로 실행
  - 진단 완료 후 마인드맵 생성 플로우로 이동 가능

- **마인드맵 사이드바**
  - 마인드맵 워크스페이스 내 사이드바에서 실행
  - 진단 결과를 추천 인벤토리에 추가

#### 4.1.2 진단 프로세스
1. **기업 선택**
   - 희망 직무를 모집한 전적이 있는 기업 목록 제공
   - 사용자가 기업 선택

2. **직무 선택**
   - 선택한 기업의 직무 목록 제공
   - 사용자가 직무 선택

3. **질문 답변**
   - 선택한 기업/직무의 기출 자기소개서 문항 제공
   - 각 질문에 대해 "가능함" 또는 "모르겠음"으로 답변
   - 모든 질문에 답변 완료 시 다음 단계로 진행

4. **진단 결과**
   - "모르겠음"으로 답변한 질문들을 기반으로 역량 빈틈 키워드 도출
   - 역량 빈틈 키워드를 추천 인벤토리에 자동 추가

### 4.2 진단 결과 관리 정책

#### 4.2.1 추천 인벤토리
- 진단 결과로 도출된 역량 빈틈 키워드를 추천 인벤토리에 저장
- 마인드맵 사이드바에서 추천 인벤토리 조회 가능
- 마인드맵 캔버스에 상시적으로 표시
- 추천 인벤토리 항목은 수동으로 삭제 가능

#### 4.2.2 진단 결과 활용
- 진단 결과는 자동으로 저장됨
- 마인드맵 생성 시 진단 결과 활용 가능
- 별도 페이지에서 진단 완료 후 "경험 정리하러 가기" 버튼으로 마인드맵 생성 플로우 이동

## 공유 및 협업 정책

### 5.1 공유 기능 정책

#### 5.1.1 공유 가능 대상
- **개인 마인드맵**: 공유 기능 없음 (개인 전용)
- **팀 마인드맵**: 공유 기능 활성화 가능

#### 5.1.2 공유 설정
- **공유 활성화**
  - 팀 마인드맵에서만 공유 기능 사용 가능
  - 프로젝트 소유자만 공유 설정 변경 가능
  - 공유 활성화 시 공유 링크 자동 생성
  - 공유 링크 형식: `https://[도메인]/mindmap/[projectId]`

- **공유 해제**
  - 프로젝트 소유자만 공유 해제 가능
  - 공유 해제 시 공유 링크 즉시 무효화
  - 기존 접근자의 접근 권한 즉시 차단

#### 5.1.3 공유 링크 관리
- 공유 링크는 프로젝트 ID 기반으로 생성
- 공유 링크는 비밀번호 없이 접근 가능
- 링크를 아는 모든 사용자가 접근 가능
- 공유 링크 노출 시 데이터 유출 위험 있음 (사용자 주의 필요)

### 5.2 접근 권한 정책

#### 5.2.1 비로그인 사용자
- **읽기 전용 모드**
  - 노드/레이아웃/설정 편집 불가
  - 노드 선택 및 내용 조회만 가능
  - "읽기 전용 모드 – 편집하려면 로그인" 배너 표시
  - STAR 작성 불가
  - 노드 생성/수정/삭제 불가

#### 5.2.2 로그인 사용자
- **프로젝트 소유자**
  - 전체 편집 가능
  - 프로젝트 설정 변경 가능
  - 공유 설정 변경 가능
  - 프로젝트 삭제 가능

- **공유 링크 접근자**
  - 편집 가능 (현재 권한 모델 단순화)
  - 노드 생성/수정/삭제 가능
  - STAR 작성 가능
  - 프로젝트 설정 변경 불가
  - 공유 설정 변경 불가
  - 프로젝트 삭제 불가

- **추후 권한 관리**
  - 역할(Role) 기반 권한 관리 도입 예정
  - 편집자, 뷰어 등 세분화된 권한 제공 예정

### 5.3 실시간 협업 정책

#### 5.3.1 동시 편집
- 여러 사용자가 동시에 마인드맵 편집 가능
- 실시간 변경 사항 동기화
- 충돌 방지 로직 적용 (최신 변경사항 우선)

#### 5.3.2 편집자 표시
- 현재 편집 중인 사용자 목록 실시간 표시
- 화면 상단에 접속자 아바타/이니셜 표시
- "나" 표시로 본인 구분
- 편집자 판단 기준: `active_editors` 테이블의 `last_seen`이 마지막 5분 이내인 사용자
- 5분 이상 비활성 시 자동으로 편집자 목록에서 제거

#### 5.3.3 협업 제약사항
- 동시 편집 시 충돌 가능성 있음
- 최신 변경사항이 우선 적용됨
- 실시간 동기화 지연 가능성 있음

## 에피소드 보관함 정책

### 6.1 조회 정책

#### 6.1.1 조회 범위
- 모든 마인드맵 프로젝트의 에피소드 조회 가능
- 개인 마인드맵과 팀 마인드맵을 탭으로 구분하여 조회
- 각 프로젝트별로 에피소드를 그룹화하여 표시

#### 6.1.2 필터 기능
- **탭 필터**: 전체 / 개인 마인드맵 / 팀 마인드맵
- **프로젝트 필터**: 드롭다운으로 특정 프로젝트만 조회
- **카테고리 필터**: 배지 타입(인턴, 동아리, 프로젝트 등)별 필터링
- **역량 태그 필터**: 에피소드에 태그된 역량 키워드로 필터링
- **검색**: 에피소드명, 경험명, STAR 내용으로 검색

#### 6.1.3 표시 정보
- 에피소드 카드: 카테고리 배지, 경험명, 에피소드명
- 기간 정보: 경험 시작일 및 종료일 (없으면 "-" 또는 "진행중")
- STAR 항목: Situation, Task, Action, Result 각 항목별 내용
- 강점/역량: 에피소드에 태그된 역량 키워드
- 작업 버튼: 편집 가능한 에피소드에만 편집 버튼 표시

### 6.2 편집 정책

#### 6.2.1 편집 뷰
- 3단 레이아웃 편집 뷰 제공
- 왼쪽: 에피소드 목록 (프로젝트별 그룹화, 클릭하여 선택)
- 중앙: STAR 입력 필드 (Situation, Task, Action, Result)
- 오른쪽: 강점/역량 태그 관리 및 저장/취소 버튼

#### 6.2.2 STAR 편집
- 각 항목별 텍스트 영역으로 자유롭게 입력
- 저장 버튼 클릭 시 데이터베이스에 저장
- 취소 버튼 클릭 시 변경사항 취소 및 편집 모드 종료

#### 6.2.3 기간 정보 편집
- 달력 컴포넌트로 시작일/종료일 선택
- 날짜 범위 선택 기능
- "진행중" 표시 (종료일이 없는 경우)
- 경험 노드의 기간 정보로 저장

#### 6.2.4 태그 관리
- 태그 추가 다이얼로그에서 역량 키워드 선택
- 선택된 태그 표시 및 제거
- 태그는 역량 키워드 목록에서만 선택 가능

## 검색 및 필터 정책

### 7.1 검색 정책

#### 7.1.1 검색 범위
- 노드 내용에 포함된 키워드 검색
- 모든 노드 타입 검색 가능 (중심 노드, 경험 카테고리, 경험, 에피소드)
- 대소문자 구분 없음
- 부분 일치 검색

#### 7.1.2 검색 결과 표시
- 검색된 노드가 하이라이트 표시
- 검색 결과 드롭다운에 노드 경로 표시
- 검색 결과 클릭 시 해당 노드로 이동 및 선택

#### 7.1.3 검색 제약사항
- 최소 검색어 길이: 1자
- 최대 검색어 길이: 제한 없음
- 특수문자 검색 가능

### 7.2 역량 필터 정책

#### 7.2.1 필터 대상
- 에피소드의 STAR 태그 기반 역량 필터링
- 에피소드 노드만 필터링 대상

#### 7.2.2 필터 결과 표시
- 필터링된 노드가 하이라이트 표시
- 필터링된 노드의 상위 노드(경험, 경험 카테고리)도 함께 표시
- 필터 해제 시 하이라이트 제거

#### 7.2.3 필터 제약사항
- 역량 키워드 목록에서만 선택 가능
- 여러 역량 태그 동시 선택 가능 (OR 조건)
- 역량 태그가 없는 에피소드는 필터링되지 않음

## 레이아웃 및 시각화 정책

### 8.1 레이아웃 정책

#### 8.1.1 레이아웃 종류
- **원형 레이아웃**
  - 중심 노드를 기준으로 원형 배치
  - 계층 구조에 따라 반경 증가

- **트리형 레이아웃**
  - 좌우 대칭 트리 구조 배치
  - 중심 노드를 기준으로 좌우 분기

- **브랜치 레이아웃**
  - 중심에서 좌우로 뻗어나가는 직각 브랜치 배치
  - 수평 확장 후 수직 배치

#### 8.1.2 레이아웃 변경
- 설정 다이얼로그에서 레이아웃 선택
- 레이아웃 변경 시 자동 재배치
- 노드 위치는 레이아웃에 따라 자동 계산
- 사용자가 수동으로 조정한 노드 위치는 레이아웃 변경 시 초기화됨

#### 8.1.3 레이아웃 제약사항
- 레이아웃 변경은 프로젝트 전체에 적용
- 개별 노드별 레이아웃 설정 불가
- 레이아웃 변경 시 노드 간 거리 자동 조정

### 8.2 시각화 정책

#### 8.2.1 그리드 표시
- 마인드맵 캔버스에 격자선 표시/숨기기 가능
- 시각적 정렬을 돕는 보조 도구
- 그리드 표시 여부는 사용자 설정에 저장

#### 8.2.2 색상 테마
- 색상 테마 선택 가능 (기본, 파스텔, 비비드, 모노크롬)
- 색상 테마는 프로젝트 전체에 적용
- 색상 테마 변경은 설정 다이얼로그에서 가능

#### 8.2.3 브랜치별 색상
- 경험 노드를 기준으로 하위 노드들에 자동 색상 할당
- 같은 경험 브랜치의 노드들은 동일한 색상 사용
- 색상은 색상 팔레트에서 자동 선택
- 사용자가 직접 색상 선택 불가

## 탭 관리 정책

### 9.1 탭 생성 정책

#### 9.1.1 새 탭 추가
- 새 탭 추가 버튼(+) 클릭 시 새 탭 패널 열림
- 새 탭 패널에서 프로젝트 열기 또는 새 프로젝트 생성 가능
- 새 탭은 기존 탭과 독립적으로 동작

#### 9.1.2 탭 타입
- **프로젝트 탭**: 기존 프로젝트를 열어 표시
- **새 탭**: 새 프로젝트 생성 또는 프로젝트 열기 패널

#### 9.1.3 탭 제약사항
- 탭 개수 제한: 없음 (현재)
- 동일 프로젝트를 여러 탭에서 열기 가능
- 각 탭은 독립적인 상태 관리

### 9.2 탭 전환 정책

#### 9.2.1 탭 전환
- 탭 클릭 시 해당 탭으로 전환
- 탭 전환 시 현재 탭의 상태 저장
- 전환된 탭의 상태 복원

#### 9.2.2 상태 관리
- 각 탭별로 독립적인 상태 관리
- 프로젝트, 노드, 선택된 노드, 포커스 노드 등 상태 저장
- 탭 전환 시 상태 즉시 복원

### 9.3 탭 닫기 정책

#### 9.3.1 탭 닫기
- 탭 닫기 버튼(X) 클릭 시 탭 닫기
- 닫힌 탭의 상태는 삭제됨
- 프로젝트 데이터는 유지 (탭만 닫힘)

#### 9.3.2 마지막 탭 처리
- 마지막 탭 닫기 시 새 탭 패널 자동 표시
- 모든 탭 닫기 시 빈 워크스페이스 표시

### 9.4 탭 복원 정책

#### 9.4.1 자동 복원
- 새로고침 후에도 열려있던 탭 자동 복원
- 로컬 스토리지에 탭 상태 저장
- 브라우저 세션 유지 시 탭 상태 복원

#### 9.4.2 복원 제약사항
- 삭제된 프로젝트의 탭은 복원되지 않음
- 로컬 스토리지 데이터 손실 시 탭 복원 불가

## 데이터 저장 및 동기화 정책

### 10.1 자동 저장 정책

#### 10.1.1 저장 트리거
- 노드 생성/수정/삭제 시 자동 저장
- STAR 작성/수정 시 저장 버튼 클릭 시 저장
- 프로젝트 설정 변경 시 자동 저장
- 레이아웃 변경 시 자동 저장

#### 10.1.2 저장 주기
- 즉시 저장 (debounce 적용, 약 500ms 지연)
- 사용자가 명시적으로 저장 버튼 클릭 불필요
- 저장 상태 표시 (저장 중, 저장 완료, 저장 실패)

### 10.2 데이터 동기화 정책

#### 10.2.1 실시간 동기화
- 팀 마인드맵에서 여러 사용자의 변경사항 실시간 동기화
- Supabase Realtime을 통한 실시간 업데이트
- 변경사항은 즉시 다른 사용자에게 반영

#### 10.2.2 동기화 충돌 처리
- 동시 편집 시 충돌 가능성 있음
- 최신 변경사항이 우선 적용됨
- 충돌 발생 시 사용자에게 알림 표시

### 10.3 데이터 백업 정책

#### 10.3.1 자동 백업
- 현재 자동 백업 기능 없음
- Supabase 데이터베이스에 자동 저장

#### 10.3.2 수동 백업
- 사용자가 내보내기 기능을 통해 직접 백업 권장
- 이미지(PNG) 또는 PDF 형식으로 저장 가능
- 데이터 손실 방지를 위한 정기적 백업 권장

## 기능별 제약사항 및 제한

### 12.1 프로젝트 제약사항

#### 12.1.1 프로젝트 개수
- 사용자당 생성 가능한 프로젝트 개수: 제한 없음 (현재)
- 추후 유료 플랜 도입 시 제한 가능

#### 12.1.2 프로젝트 크기
- 노드 개수 제한: 없음 (현재)
- 프로젝트 파일 크기 제한: 없음 (현재)
- 성능 저하 시 노드 개수 제한 도입 가능

### 12.2 노드 제약사항

#### 12.2.1 노드 개수
- 프로젝트당 노드 개수 제한: 없음 (현재)
- 계층 깊이 제한: 없음 (현재)
- 성능 저하 시 제한 도입 가능

#### 12.2.2 노드 내용
- 노드 이름 최대 길이: 200자
- 노드 내용은 텍스트만 지원 (이미지, 링크 등 미지원)
- 특수문자 사용 가능

### 12.3 STAR 제약사항

#### 12.3.1 STAR 항목
- 각 항목별 최대 길이: 제한 없음
- 필수 입력 항목 없음
- 빈 STAR 저장 가능

#### 12.3.2 역량 태그
- 에피소드당 태그 개수 제한: 없음
- 태그는 역량 키워드 목록에서만 선택 가능
- 직접 입력 불가

### 12.4 공유 제약사항

#### 12.4.1 공유 링크
- 공유 링크는 비밀번호 없이 접근 가능
- 링크 노출 시 데이터 유출 위험
- 공유 링크 만료 기능 없음

#### 12.4.2 동시 편집
- 동시 편집자 수 제한: 없음 (현재)
- 성능 저하 시 제한 도입 가능
- 동시 편집 시 충돌 가능성 있음

### 12.5 내보내기 제약사항

#### 12.5.1 이미지 내보내기
- PNG 형식만 지원
- 해상도 제한: 없음 (현재)
- 대용량 마인드맵의 경우 내보내기 시간 증가 가능
