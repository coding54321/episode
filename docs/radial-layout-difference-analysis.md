# 원형 레이아웃 차이점 분석

## 레퍼런스 이미지 분석

### 1. 중심 노드의 자식 배치
- **왼쪽**: 2-3개 노드가 위에서 아래로 세로 배치
- **오른쪽**: 2-3개 노드가 위에서 아래로 세로 배치
- **배치 방식**: 완전히 좌우로만 배치 (X축 기준)

### 2. 분기 노드의 자식 배치
- **방향**: 각 분기 노드가 중심 노드로부터 뻗어나간 방향을 따라 **직선적으로** 배치
- **예시**: 
  - 왼쪽 위 분기 → 자식들이 왼쪽 위 방향으로 직선 배치
  - 오른쪽 아래 분기 → 자식들이 오른쪽 아래 방향으로 직선 배치
- **여러 자식**: 그 방향을 중심으로 좌우로 약간 분산 (작은 각도)

### 3. 반경/간격
- **일정한 간격**: 각 레벨마다 일정한 거리로 뻗어나감
- **직선적**: 원형이 아닌 직선적으로 뻗어나가는 형태

## 현재 구현 분석

### 1. 중심 노드의 자식 배치
- ✅ **좌우 배치**: 왼쪽과 오른쪽으로 분리 배치
- ✅ **세로 분산**: 각 방향에서 위/아래로 분산
- ✅ **왼쪽에 더 많이**: 이미지처럼 배치

### 2. 분기 노드의 자식 배치
- ⚠️ **원형 배치**: `cos(angle) * radius`, `sin(angle) * radius` 사용
- ⚠️ **부모 노드 기준**: 부모 노드를 중심으로 원형 배치
- ⚠️ **반경 증가**: 레벨이 깊을수록 반경이 증가

### 3. 문제점
- **원형 vs 직선**: 현재는 원형으로 배치되지만, 레퍼런스는 직선적으로 뻗어나감
- **부모 기준 vs 중심 기준**: 현재는 부모 노드를 기준으로 배치하지만, 레퍼런스는 중심 노드 방향을 따라 직선 배치
- **반경 증가**: 레벨마다 반경이 증가하지만, 레퍼런스는 일정한 간격

## 주요 차이점 요약

| 구분 | 레퍼런스 이미지 | 현재 구현 |
|------|---------------|----------|
| **중심 노드 자식** | 좌우 세로 배치 | ✅ 동일 |
| **분기 노드 자식** | 중심 방향으로 직선 배치 | ⚠️ 부모 기준 원형 배치 |
| **반경** | 일정한 간격 | ⚠️ 레벨마다 증가 |
| **배치 형태** | 직선적 가지 뻗어나가기 | ⚠️ 원형 방사형 |

## 수정 필요 사항

1. **분기 노드 자식 배치**: 원형이 아닌 직선적으로 배치
2. **반경 계산**: 레벨마다 증가하지 않고 일정한 간격 유지
3. **방향 유지**: 중심 노드로부터의 방향을 따라 직선적으로 배치
