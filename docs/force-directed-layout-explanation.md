# Force-Directed Layout (물리 시뮬레이션) 설명

## 개념

Force-Directed Layout은 **물리학의 힘(Force)을 시뮬레이션**하여 노드들을 배치하는 알고리즘입니다. 마치 노드들이 서로 끌어당기고 밀어내는 것처럼 동작합니다.

## 작동 원리

### 1. 인력 (Attraction Force)
- **연결된 노드들**은 서로 끌어당깁니다
- 부모-자식 관계가 있는 노드들이 가까워지려고 합니다
- 연결선의 길이가 일정한 거리(ideal distance)를 유지하려고 합니다

### 2. 반발력 (Repulsion Force)
- **모든 노드들**은 서로 밀어냅니다
- 노드들이 겹치지 않도록 일정한 거리를 유지합니다
- 너무 가까이 있으면 서로 밀어냅니다

### 3. 시뮬레이션 과정
1. 초기 위치 설정 (랜덤 또는 기존 위치)
2. 각 노드에 작용하는 힘 계산
3. 힘에 따라 노드 위치 업데이트
4. 반복하여 안정적인 위치 찾기

## 장점

✅ **유기적인 배치**: 자연스럽고 유연한 레이아웃
✅ **복잡한 관계 표현**: 노드 간 관계를 시각적으로 잘 보여줌
✅ **자동 최적화**: 노드 개수와 관계에 따라 자동으로 최적 배치
✅ **동적 조정**: 노드 추가/삭제 시 자연스럽게 재배치

## 단점

⚠️ **계산 비용**: 많은 반복 계산이 필요 (성능 이슈)
⚠️ **결과 예측 어려움**: 매번 약간씩 다른 배치
⚠️ **안정화 시간**: 최적 위치를 찾기까지 시간 소요

## 시각적 예시

```
초기 상태 (랜덤):
    A  B
  C    D
    E

시뮬레이션 후 (안정화):
      A
     / \
    B   C
   / \
  D   E
```

## 다른 레이아웃과의 비교

| 레이아웃 | 배치 방식 | 예측 가능성 | 성능 |
|---------|---------|-----------|------|
| **Radial** | 중심 기준 원형 | 높음 | 빠름 |
| **Hierarchical** | 위-아래 계층 | 높음 | 빠름 |
| **Tree** | 좌우 대칭 트리 | 높음 | 빠름 |
| **Force-Directed** | 물리 힘 시뮬레이션 | 낮음 | 느림 |

## 구현 복잡도

Force-Directed Layout은 다음과 같은 알고리즘을 사용합니다:
- **Fruchterman-Reingold**: 인기 있는 알고리즘
- **Barnes-Hut**: 대규모 그래프에 최적화
- **Spring-Force**: 스프링처럼 동작하는 힘 모델

## 사용 사례

- **소셜 네트워크**: 사람들 간의 관계 시각화
- **의존성 그래프**: 코드/모듈 간 의존성 표현
- **지식 그래프**: 개념 간 복잡한 관계 표현
- **마인드맵**: 자유로운 아이디어 연결
