# Mermaid AI 플로우차트 프롬프트 (분기점별 세분화)

## 사용 방법
각 프롬프트는 독립적으로 Mermaid AI에 입력하여 플로우차트를 생성할 수 있습니다. 
여러 프롬프트를 순차적으로 생성한 후, 필요에 따라 연결하여 전체 플로우를 완성하세요.

---

## 📍 진입점 분기

### 프롬프트 1-1: 서비스 진입점 및 초기 분기
```
Episode 서비스의 진입점과 초기 분기를 flowchart로 그려줘.

진입점은 두 가지:
1. 랜딩 페이지 (/)
2. 공유 링크 (/share/[nodeId])

랜딩 페이지에서:
- 로그인 상태 확인
  - 로그인 안됨 → 서비스 소개 표시 → 기능 소개 페이지 이동 가능 (마인드맵, 공백 진단, 아카이브) → 시작하기 → 로그인 페이지
  - 로그인 됨 → 프로젝트 목록 페이지로 리다이렉트
- OAuth 코드가 URL에 있으면 → OAuth 콜백 페이지로 리다이렉트

공유 링크에서:
- 별도 플로우 (프롬프트 5-1 참조)

각 분기점을 다이아몬드로, 페이지를 둥근 사각형으로 표시해줘.
```

## 🔐 인증 플로우

### 프롬프트 2-1: 로그인 페이지 진입 및 상태 확인
```
로그인 페이지 (/login)의 초기 상태 확인 플로우를 flowchart로 그려줘.

로그인 페이지 진입:
- 로그인 상태 확인
  - 이미 로그인됨:
    - returnUrl 있음 → returnUrl로 이동
    - returnUrl 없음:
      - 프로젝트 있음 → 프로젝트 목록 (/mindmaps)
      - 프로젝트 없음 → 배지 선택 페이지 (/badge-selection)
  - 로그인 안됨 → 카카오 로그인 버튼 표시

조건 분기를 명확히 표시해줘.
```

### 프롬프트 2-2: 카카오 OAuth 인증 프로세스
```
카카오 OAuth 인증 프로세스를 flowchart로 그려줘.

카카오 로그인 버튼 클릭:
- 카카오 OAuth 페이지로 리다이렉트
- 카카오 계정으로 로그인 및 동의
- 카카오 로그인 완료
- OAuth 콜백 (/auth/callback?code=...)으로 리다이렉트

OAuth 콜백 처리:
- 인증 코드 처리
- Supabase Auth로 사용자 정보 조회/생성
  - 성공 → localStorage에 사용자 정보 저장
    - returnUrl 있음 → returnUrl로 이동
    - returnUrl 없음:
      - 프로젝트 있음 → 프로젝트 목록
      - 프로젝트 없음 → 배지 선택 페이지
  - 실패 → 에러 메시지 표시 → 로그인 페이지 유지

에러 케이스를 빨간색으로 표시해줘.
```

### 프롬프트 2-3: 로그아웃 프로세스
```
로그아웃 프로세스를 flowchart로 그려줘.

어떤 페이지에서든:
- 로그아웃 버튼 클릭
- Supabase Auth 세션 종료
- localStorage의 사용자 정보 삭제
- 로그인 페이지로 리다이렉트

간단한 선형 플로우로 표시해줘.
```

## 📁 프로젝트 관리 플로우

### 프롬프트 3-1: 프로젝트 목록 페이지
```
프로젝트 목록 페이지 (/mindmaps)의 플로우를 flowchart로 그려줘.

프로젝트 목록 페이지 진입:
- 로그인 확인
  - 로그인 안됨 → 로그인 페이지로 리다이렉트
  - 로그인 됨 → 프로젝트 목록 조회
    - 프로젝트 있음:
      - 프로젝트 카드 표시 (프로젝트명, 설명, 노드 개수, 최근 수정일)
      - 프로젝트 클릭 → 마인드맵 편집 페이지 (/mindmap/[projectId])
      - 즐겨찾기 토글 → 즐겨찾기 상태 업데이트
      - 새 프로젝트 버튼 클릭 → 배지 선택 페이지
    - 프로젝트 없음:
      - 빈 상태 표시
      - 새 프로젝트 버튼 표시 → 클릭 → 배지 선택 페이지

프로젝트 카드와 버튼 액션을 명확히 구분해줘.
```

### 프롬프트 3-2: 배지 선택 및 프로젝트 생성
```
배지 선택 페이지 (/badge-selection)부터 프로젝트 생성까지의 플로우를 flowchart로 그려줘.

배지 선택 페이지 진입:
- 로그인 확인
  - 로그인 안됨 → 로그인 페이지로 리다이렉트
  - 로그인 됨 → 8가지 배지 표시 (인턴, 학업, 동아리, 프로젝트, 아르바이트, 봉사활동, 공모전, 기타)

배지 선택:
- 배지 선택 (최소 1개 이상 필수)
- 기타 배지 선택 시 → 커스텀 라벨 입력 필드 표시
- 완료 버튼 클릭

프로젝트 초기화:
- UUID 프로젝트 ID 생성
- 프로젝트명 자동 생성: '{사용자명}의 경험 맵'
- 중앙 노드 생성 (사용자명 또는 '나')
- 선택한 배지 수만큼 대분류 노드 생성 (원형 배치)
- 각 배지 노드를 중앙 노드의 자식으로 연결
- Supabase DB에 프로젝트 정보 저장
- 마인드맵 편집 페이지로 이동

각 단계를 순차적으로 표시해줘.
```

## 🗺️ 마인드맵 편집 플로우

### 프롬프트 4-1: 마인드맵 편집 페이지 진입 및 온보딩
```
마인드맵 편집 페이지 (/mindmap/[projectId])의 진입 및 온보딩 플로우를 flowchart로 그려줘.

마인드맵 편집 페이지 진입:
- 로그인 확인
  - 로그인 안됨 → 로그인 페이지로 리다이렉트
  - 로그인 됨 → 프로젝트 데이터 로드
    - 프로젝트 없음 → 에러 메시지 → 프로젝트 목록으로 리다이렉트
    - 프로젝트 있음 → 노드 데이터 로드
      - 최초 진입 → 온보딩 튜토리얼 표시 여부 확인
        - 튜토리얼 미표시 → 4단계 튜토리얼 오버레이 표시
          - 건너뛰기 → 튜토리얼 표시 여부 저장 → 마인드맵 표시
          - 완료 → 튜토리얼 표시 여부 저장 → 마인드맵 표시
        - 튜토리얼 이미 표시됨 → 마인드맵 표시

온보딩 튜토리얼 4단계:
1. 계층 구조로 경험 정리하기
2. 공백 진단으로 부족한 경험 찾기
3. 어시스턴트로 노드 확장하기
4. 캔버스 조작과 노드 편집

튜토리얼 단계를 명확히 표시해줘.
```

### 프롬프트 4-2: 노드 기본 조작 (클릭, 더블클릭, 드래그)
```
마인드맵에서 노드의 기본 조작 플로우를 flowchart로 그려줘.

노드 클릭:
- 노드 선택 상태로 변경 (시각적 강조)
- 선택된 노드의 상세 정보를 사이드바에 표시
- AI 어시스턴트 자동 표시 (STAR 작성 시작)
- 다른 노드 클릭 시 → 이전 노드 선택 해제 및 새 노드 선택

노드 더블클릭:
- 인라인 편집 모드로 전환
- 라벨 텍스트 수정
- Enter 키 또는 포커스 아웃 → 저장
- Supabase DB에 저장
- 변경 사항 즉시 반영

노드 드래그:
- 노드 위치 이동
- 드롭 시 새로운 좌표(x, y) 저장
- Supabase DB에 저장
- 변경 사항 즉시 반영
- 부모-자식 관계는 유지 (계층 구조 보존)

각 액션을 별도 경로로 표시해줘.
```

### 프롬프트 4-3: 노드 우클릭 컨텍스트 메뉴
```
마인드맵 노드의 우클릭 컨텍스트 메뉴 플로우를 flowchart로 그려줘.

노드 우클릭:
- 컨텍스트 메뉴 표시
  - 하위 노드 추가:
    - 방향 선택 (오른쪽, 왼쪽, 위, 아래)
    - 새 노드 생성 (레벨에 따라 노드 타입 자동 설정)
    - 기본 라벨 자동 생성
    - Supabase DB에 저장
    - 즉시 편집 모드로 전환
  - 편집:
    - 인라인 편집 모드로 전환
  - STAR 정리하기:
    - 사이드바 STAR 탭 열기
  - 공유하기:
    - 공유 링크 생성 플로우 (프롬프트 5-2 참조)
  - 공유 해제:
    - 공유 해제 확인 다이얼로그 표시
    - 확인 → shared_nodes 테이블에서 공유 정보 삭제
    - 노드의 공유 상태 제거
  - 새 탭에서 열기:
    - 새 탭 생성
    - 해당 노드를 중심으로 마인드맵 표시
  - 삭제:
    - 삭제 확인 다이얼로그 표시 (하위 노드 포함 삭제 안내)
    - 확인 → 해당 노드 및 모든 하위 노드 삭제
    - 관련 STAR 에셋도 함께 삭제
    - Supabase DB에 반영
    - 변경 사항 즉시 반영

각 메뉴 항목을 별도 분기로 표시해줘.
```

### 프롬프트 4-4: 노드 추가 (마우스 오버 및 태그 드롭)
```
마인드맵에서 노드를 추가하는 두 가지 방법의 플로우를 flowchart로 그려줘.

방법 1: 마우스 오버로 추가
- 노드에 마우스 오버
- '+' 버튼 표시
- '+' 버튼 클릭
- 방향 선택 (오른쪽, 왼쪽, 위, 아래)
- 새 노드 생성
- 즉시 편집 모드로 전환

방법 2: 태그 드롭으로 추가
- 공백 진단의 추천 인벤토리에서 역량 태그를 마인드맵 노드로 드래그
- 노드 추가 다이얼로그 표시
  - 노드 층위에 따른 안내 문구 표시
  - 관련 역량 태그 정보 표시
  - 노드 이름 입력 필드 제공
  - 생성하기 버튼 클릭:
    - 새 노드 생성
    - AI 어시스턴트 자동 열기
    - 새 노드 선택
  - 취소 버튼 클릭:
    - 다이얼로그 닫기

두 가지 방법을 병렬로 표시해줘.
```

### 프롬프트 4-5: 캔버스 조작 (이동 및 줌)
```
마인드맵 캔버스 조작 플로우를 flowchart로 그려줘.

캔버스 이동:
- Ctrl/Cmd 키를 누른 상태에서 드래그
- 캔버스 위치 정보 저장 (viewport 좌표)

줌 인/아웃:

- 줌 레벨 표시 (선택사항)
- 최소/최대 줌 레벨 제한
- 부드러운 줌 애니메이션 제공

간단한 선형 플로우로 표시해줘.
```

## 🎯 사이드바 기능 플로우

### 프롬프트 4-6: 공백 진단하기 탭 (부족 역량확인)
```
사이드바의 공백 진단하기 탭 중 '부족 역량확인' 서브 탭의 플로우를 flowchart로 그려줘.

공백 진단하기 버튼 클릭:
- 사이드바 열기
- 공백 진단 탭 활성화
- 부족 역량확인 서브 탭 표시

기업 및 직무 선택:
- 기업 목록에서 기업 선택 (삼성전자, 현대자동차, KIA 등)
- 선택한 기업의 직무 목록 표시
- 직무 선택 (SW개발, 연구개발, 마케팅 등)
- 선택 완료 → 해당 직무의 문항 목록 표시

문항 체크:
- 선택한 직무의 최근 5년간 문항 목록 표시
- 각 문항에 대해 답변 어려움 정도 선택 (쉬움/보통/어려움)
- 어려움 선택 → 해당 문항의 역량 태그 추출 → 추천 인벤토리에 자동 추가
- 다음 문항으로 자동 이동 또는 수동 이동
- 모든 문항 완료 → 분석 결과 표시
  - 어려운 문항 개수 및 부족한 역량 개수 표시
  - 추천 인벤토리에 추가된 역량 목록 표시
  - 결과 확인 후 마인드맵으로 이동하여 부족한 경험 추가 안내

단계별로 순차적으로 표시해줘.
```

### 프롬프트 4-7: 공백 진단하기 탭 (추천 인벤토리)
```
사이드바의 공백 진단하기 탭 중 '추천 인벤토리' 서브 탭의 플로우를 flowchart로 그려줘.

추천 인벤토리 탭:
- 공백 진단 결과로 추출된 역량을 추천 인벤토리에 추가
- 역량 태그 목록 조회
- 역량 태그를 마인드맵 노드로 드래그:
  - 노드 추가 다이얼로그 표시 (프롬프트 4-4 참조)
- 역량 태그 삭제:
  - 삭제 버튼 클릭
  - 역량 태그 제거
- 역량 태그 클릭:
  - 연결된 문항 목록 조회

드래그 앤 드롭 액션을 명확히 표시해줘.
```

### 프롬프트 4-8: 어시스턴트 탭 (AI 챗봇)
```
사이드바의 어시스턴트 탭 플로우를 flowchart로 그려줘.

어시스턴트 버튼 클릭:
- 사이드바 열기
- 어시스턴트 탭 활성화

또는 노드 선택 시:
- AI 어시스턴트 자동 표시

AI 챗봇 대화:
- 노드 타입과 레벨에 따라 적절한 시작 질문 제공
- 대화형 인터페이스로 사용자와 상호작용
- 대화 내역을 채팅 형식으로 표시 (사용자/AI 구분)

STAR 단계별 대화 진행:
- Situation(상황) 단계:
  - 상황에 대한 질문 및 답변 수집
  - 단계 완료 시 진행 상황 시각화 (체크 표시)
- Task(과제) 단계:
  - 과제와 역할에 대한 질문 및 답변 수집
  - 단계 완료 시 진행 상황 시각화 (체크 표시)
- Action(행동) 단계:
  - 행동과 과정에 대한 질문 및 답변 수집
  - 단계 완료 시 진행 상황 시각화 (체크 표시)
- Result(결과) 단계:
  - 결과와 성과에 대한 질문 및 답변 수집
  - 단계 완료 시 진행 상황 시각화 (체크 표시)

모든 단계 완료:
- STAR 초안 생성 버튼 활성화
- STAR 초안 생성 버튼 클릭:
  - 대화 내용을 기반으로 STAR 초안 자동 생성
  - 생성된 초안을 STAR 에디터에 자동 로드
  - 생성 완료 알림 표시

사용자 메시지 입력:
- 사용자 메시지 입력
- AI 응답 생성 (키워드 기반 또는 AI API)
- 응답 생성 중 로딩 표시 (타이핑 애니메이션)
- 생성된 응답을 채팅 형식으로 표시
- 응답 내용을 STAR 데이터에 자동 저장

STAR 단계를 순차적으로 표시하고, 각 단계 완료를 명확히 표시해줘.
```
